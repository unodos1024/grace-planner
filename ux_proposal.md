# 제자훈련 영적 습관 관리 앱: Grace Planner (그레이스 워크) UX/UI 기획안

## 1. UX 설계 아이덴티티
- **따뜻한 동행**: 차가운 생산성 앱이 아닌, 따뜻한 신앙의 동반자 같은 느낌.
- **최소한의 입력**: 훈련생의 피로도를 줄이기 위해 '체크'와 '스위치' 중심의 UX.
- **가시적인 성장**: 90일 통독, 주간 QT 달성률 등을 시각화하여 성취감 고취.

---

## 2. 주요 화면 구조 (Information Architecture)

### 1) 홈 화면 (Home: Today's Grace)
- **상단 헤더**: 오늘의 날짜 + "오늘도 주님과 함께 한 걸음씩!" (격려 문구)
- **주간 현황 카드**: 이번 주 기도/QT/통독 달성률 요약.
- **주간 캘린더**: 월~일 요일 표시, 오늘 강조.
- **오늘의 과제 (Task List)**:
  - 교재 예습 (Simple Check)
  - 성경일기 (Check + Progress Bar)
  - 오늘의 QT (Check)
  - 20분 기도 (Timer Button)
  - 성경 암송 (Card Preview)
  - 독서/설교요약 (Status)

### 2) 기도 타이머 화면 (Immersive Prayer)
- **UI**: 중앙에 부드러운 원형 인디케이터.
- **기능**: 시작/일시정지/종료 버튼. 20분 도달 시 자동 완료 처리.
- **분위기**: 차분한 배경 이미지 또는 그라데이션.

### 3) 성경암송 화면 (Flashcards)
- **UI**: 옆으로 넘기는 카드 형태 (Swipe).
- **상태 관리**: [읽음] -> [암기 중] -> [완료] 3단계 버튼.
- **데이터**: 32주 차별 고정 데이터 바인딩.

### 4) 기록장 (독서 & 설교요약)
- **독서**: 연간 14권 리스트, 책 표지 형태의 UI, 독후감 가이드 제공.
- **설교**: 날짜별 리스트, 예배 종류 선택 필터, 텍스트 요약 기록.

---

## 3. 기술 및 데이터 설계

### Backend/DB
- **Base**: .NET Web API + Oracle Cloud ATP
- **Storage**: 개인 기록은 DB 저장, 고정 데이터(암송 구절 등)는 JSON Seed 파일 관리.

### 데이터 모델 (Entity)
- **DailyTask**: 매일의 수행 여부 기록.
- **BibleReading90**: 90일 통독 진척도 관리.
- **PrayerLog**: 기도 시간(분 단위) 기록.
- **SermonNote**: 설교 요약 (CLOB 타입).
