# Grace Planner: UX/UI Design Proposal & Flow

제자훈련의 여정을 돕는 동반자, **Grace Planner**의 상세 설계안입니다.

## 1. 주요 화면 흐름 (Main Flow)

앱의 흐름은 훈련생이 하루 중 언제든지 접속하여 자신의 영적 상태를 점검하고, 필요한 과제를 즉시 수행할 수 있도록 선순환 구조로 설계되었습니다.

1.  **홈 (오늘의 은혜)**: 앱 진입 시 현재의 영적 온도(주간 달성률)와 오늘 해야 할 과제를 한눈에 확인합니다.
2.  **과제 수행 (Action)**: 
    *   **기도**: 타이머 화면으로 진입하여 깊은 기도의 시간을 갖습니다.
    *   **암송**: 플래시 카드를 통해 말씀을 마음에 새깁니다.
    *   **QT/통독**: 읽기 완료 후 심플하게 체크하여 성취감을 얻습니다.
3.  **기록 (Record)**: 설교 요약이나 독후감을 작성하며 배운 내용을 정리합니다.
4.  **설정 (Identity)**: 자신의 알림 시간과 훈련 목표를 조정합니다.

---

## 2. 핵심 화면 UI 구조 (Core UI Structure)

### ① 홈 화면 (Home Screen)
- **Visual Identity**: 파스텔 민트(#A8E6CF)와 베이지 색감을 사용하여 심리적 안정감을 제공합니다.
- **주간 캘린더 스트립**: iOS 스타일의 수평 캘린더로 오늘이 전체 여정 중 어디에 있는지 시각화합니다.
- **카드형 과제 리스트**: 각 과제를 독립된 카드로 배치하여 복잡도를 낮추고 클릭(터치) 영역을 확보합니다.

### ② 기도 타이머 (Prayer Timer)
- **UX Intent**: 스마트폰의 방해 요소를 최소화하기 위해 '몰입형 화면'으로 구성합니다.
- **Design**: 부드러운 원형 인디케이터와 차분한 배경음을 옵션으로 제공할 수 있는 구조입니다.

### ③ 성경 암송 플래시 카드 (Memory Verse)
- **UX Intent**: '암기'라는 부담감을 줄이기 위해 게임화(Gamification) 요소를 미세하게 가미합니다.
- **Interaction**: 카드를 뒤집는(Flip) 애니메이션과 상태 버튼(읽음/암기중/완료)을 통해 진행 상황을 즉각 반영합니다.

---

## 3. UX 설계 의도 (Design Rationale)

1.  **"감시가 아닌 격려"**: 데이터가 쌓이는 과정을 '성장하는 나무'나 '채워지는 물병' 같은 비유적 UI로 표현하여 사용자에게 긍정적인 피드백을 줍니다.
2.  **"One-Tap Logic"**: 모든 과제 체크는 최소한의 탭으로 완료됩니다. 텍스트 입력은 설교 요약과 독서 기록 등 깊은 묵상이 필요한 시점에만 배치합니다.
3.  **"영적 오아시스"**: 복잡한 비즈니스 앱의 느낌을 배제하고, 여백의 미를 살린 레이아웃을 통해 앱 사용 자체가 하나의 묵상이 되도록 디자인했습니다.

---

## 4. 데이터 활용 계획
- **JSON Seed Data**: 성경 암송 구절과 신앙 서적 리스트는 서버에서 JSON 형태로 관리하여 관리자가 쉽게 업데이트하고 앱이 오프라인에서도 작동하도록 설계합니다.
- **RESTful API**: 훈련 내용의 무결성을 위해 `DailyTask`와 `WeeklySummary` API를 통해 서버와 실시간으로 동기화합니다.
